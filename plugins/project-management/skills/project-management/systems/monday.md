# Monday.com

- **What it is:** Project management and work OS. Optimate both uses it internally and implements it for clients.
- **Typical task patterns:**
  - Board setup: define columns → create groups → configure views → set automations
  - Custom views: table, kanban, timeline, calendar — choose based on use case
  - Automations: monday.com native automations for status changes, notifications, item creation
- **Common edge cases:**
  - Column types: choose carefully — changing column type after data exists can lose data
  - Subitems: have their own board with separate columns — don't assume parent columns exist
  - Board relations: must be pre-configured in column settings before linking items
  - Permissions: workspace vs. board level — plan access model early
